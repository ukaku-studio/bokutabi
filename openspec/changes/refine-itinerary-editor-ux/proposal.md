# Proposal: refine-itinerary-editor-ux

## Summary
旅程作成画面のUX改善。金額入力、プレビューボタン、日付タブ、パネル削除、レイアウト調整などの修正を行う。

## Motivation
- 金額欄のスピンボタンがモバイルで誤操作を招く
- プレビューボタンが固定サイズで文字が収まらない場合がある
- 日付タブに曜日がなく、どの日か分かりにくい
- フィルタ適用中にパネル追加ができない
- パネル削除機能がない
- パネル内のレイアウトが直感的でない

## Requirements

### R1: 金額入力欄のスピンボタン非表示
- `type="number"` の上下ボタン（スピンボタン）を非表示にする
- `inputMode="numeric"` を追加してモバイルで数字キーボードを表示
- CSSで `-webkit-inner-spin-button`, `-moz-appearance` を制御

### R2: プレビューボタンのレスポンシブ対応
- 固定サイズ（h-12 w-12）から幅自動調整に変更
- `px-4 py-2` + `whitespace-nowrap` で文字に応じた幅に

### R3: 日付タブに曜日追加
- 現在: `12/31` 形式
- 変更後: `12/31(火)` 形式
- 既存の `getDayOfWeek` 関数を再利用

### R4: フィルタ適用中でもパネル追加可能に
- 現在: `!selectedDateFilter` 条件でプラスボタンを非表示
- 変更: 条件を削除し、フィルタ中も追加ボタンを表示
- 追加時は選択中の日付を自動設定

### R5: パネル削除機能
- 各パネルの右上に削除ボタン（×アイコン）を配置
- 初期値と異なる場合は確認モーダルを表示
- 初期値: `{ date: '', time: '', location: '', memo: '', icon: '📍', cost: '', currency: 'JPY' }`
- 最後の1パネルは削除不可（または空の場合のみ削除可）

### R6: パネル内レイアウト変更
- 現在の順序: 日時 → 場所ボタン → 金額 → メモ
- 変更後: 日時 → メモ → 場所ボタン → 金額
- メモのプレースホルダを「プランの詳細（任意）」に変更

## Scope
- `src/pages/CreateItineraryPage.tsx`
- `src/i18n/locales/en.json`, `ja.json`
- 新規: 削除確認モーダルコンポーネント（インライン実装）

## Out of Scope
- Google Maps統合
- プレビューページのレイアウト変更
- 他ページへの影響

## Acceptance Criteria
- [ ] 金額欄でスピンボタンが表示されない
- [ ] プレビューボタンが文字幅に応じてリサイズ
- [ ] 日付タブに曜日が表示される
- [ ] フィルタ中でもパネル追加ができる
- [ ] パネル削除ができ、変更がある場合は確認される
- [ ] パネル内でメモ欄が日時の下に配置される
