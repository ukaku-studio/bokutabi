# Change: 旅程作成〜プレビューUX改善と位置検索最適化

## Why
- 作成・確認・保存までの導線が分かりづらく、保存前に入力内容が失われるケースがあったため
- 位置検索の言語・表示名・住所表示の扱いが不安定だったため
- 旅程パネルの空状態や追加挙動の一貫性が必要だったため

## What Changes
- 「内容を確認する」→プレビュー→「保存する/修正する」の導線を追加
- 位置検索の言語優先順位とフォールバック検索を導入
- 公式名がある場合の住所表示抑制、再検索時の入力値の最適化
- 旅程パネルの空状態・追加・候補選択・外観ルールの整理
- 初回アクセス時の言語/テーマをブラウザ設定に準拠

## Impact
- 影響するスペック: itinerary-ui
